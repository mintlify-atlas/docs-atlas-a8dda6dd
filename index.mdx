gstack turns Claude Code from one generic assistant into a team of specialists you can summon on demand. Eight opinionated workflow skills for planning, code review, shipping, browser automation, QA testing, and engineering retrospectives — all as slash commands.

## Why gstack?

Without gstack, Claude Code takes your requests literally. It implements exactly what you say, even when the real product is something bigger. Code review is inconsistent. Shipping becomes a long back-and-forth. The agent can write code but can't see your app — it's half blind.

With gstack, you get explicit cognitive modes:

## Quick Start

<Steps>
  <Step title="Install gstack">
    Open Claude Code and paste this command:
    
    ```bash
    git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup
    ```
  </Step>
  
  <Step title="Add to your project">
    Copy gstack to your repo so teammates get it automatically:
    
    ```bash
    cp -Rf ~/.claude/skills/gstack .claude/skills/gstack && rm -rf .claude/skills/gstack/.git && cd .claude/skills/gstack && ./setup
    ```
  </Step>
  
  <Step title="Use the skills">
    Start using slash commands in Claude Code:
    
    - `/plan-ceo-review` — Product direction
    - `/plan-eng-review` — Technical planning  
    - `/review` — Code review
    - `/ship` — Ship your branch
    - `/browse` — Browser automation
    - `/qa` — QA testing
  </Step>
</Steps>

## Example Workflow

```
You:   [enter plan mode, describe the feature]
       I want to add seller photo upload to the listing app.

You:   /plan-ceo-review

Claude: "Photo upload" is not the feature. The real job is helping sellers
        create listings that actually sell. Here's what the 10-star version
        looks like: auto-identify the product, pull specs and pricing comps...

You:   /plan-eng-review

Claude: [Architecture diagram, state machine for upload → classify → enrich
        → draft pipeline, async job boundaries, failure modes, test matrix]

You:   [exit plan mode, implement the plan]

You:   /review

Claude: Race condition: two tabs can overwrite cover-photo selection.
        Trust boundary: pulling web data creates prompt injection vector.

You:   (fix the issues)

You:   /ship

Claude: [Syncs main, runs tests, pushes branch, opens PR — 6 tool calls, done]

You:   /qa

Claude: Analyzing branch diff... 8 files changed, 3 routes affected.
        [Tests /listings/new, /listings/:id, /api/listings]
        All 3 routes working. Upload + enrichment flow passes end to end.
```

## Who This Is For

You already use Claude Code heavily and want consistent, high-rigor workflows instead of one mushy generic mode. You want to tell the model what kind of brain to use right now — founder taste, engineering rigor, paranoid review, or fast execution.

This is not a prompt pack for beginners. It is an operating system for people who ship.
