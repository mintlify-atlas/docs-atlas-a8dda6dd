# setup-browser-cookies

**Import cookies from your real browser (Comet, Chrome, Arc, Brave, Edge) into the headless browse session.** Opens an interactive picker UI where you select which cookie domains to import. Use before QA testing authenticated pages.

## When to Use

Use `/setup-browser-cookies` when you:

- **Need to test authenticated pages** with `/browse` or `/qa`
- **Don't want to manually log in** through the headless browser every time
- **Have sessions in your real browser** (GitHub, staging environments, local dev)
- **Want to test multiple domains** without repeatedly entering credentials

<Note>
**This is session manager mode.** Instead of manually logging in through the headless browser every time, import your real sessions directly.
</Note>

## How It Works

1. Find the browse binary
2. Run `cookie-import-browser` to detect installed browsers and open the picker UI
3. User selects which cookie domains to import in their browser
4. Cookies are decrypted (via macOS Keychain) and loaded into the Playwright session

## Interactive Picker

The skill opens an interactive picker UI in your default browser where you can:

- **Switch between installed browsers** (Comet, Chrome, Arc, Brave, Edge auto-detected)
- **Search domains** (e.g., type "github" to find github.com)
- **Click "+" to import** a domain's cookies
- **Click trash to remove** imported cookies

**Privacy:** Only domain names and cookie counts are shown in the UI — no cookie values are exposed.

## Direct Import (Alternative)

If you specify a domain directly, the skill skips the UI:

```bash
/setup-browser-cookies github.com
```

This imports cookies for `github.com` from the default browser (Comet, or first installed Chromium browser).

You can also specify the browser:

```bash
/setup-browser-cookies github.com --browser chrome
```

## Workflow

<Steps>
  <Step title="Find the browse binary">
    The skill runs a setup check to locate the browse binary:

    ```bash
    _ROOT=$(git rev-parse --show-toplevel 2>/dev/null)
    B=""
    [ -n "$_ROOT" ] && [ -x "$_ROOT/.claude/skills/gstack/browse/dist/browse" ] && B="$_ROOT/.claude/skills/gstack/browse/dist/browse"
    [ -z "$B" ] && B=~/.claude/skills/gstack/browse/dist/browse
    if [ -x "$B" ]; then
      echo "READY: $B"
    else
      echo "NEEDS_SETUP"
    fi
    ```

    If `NEEDS_SETUP`, it asks permission to build the binary (~10 seconds).
  </Step>
  <Step title="Open the cookie picker">
    ```bash
    $B cookie-import-browser
    ```

    This auto-detects installed Chromium browsers and opens the picker UI.

    The skill tells the user:

    **"Cookie picker opened — select the domains you want to import in your browser, then tell me when you're done."**
  </Step>
  <Step title="Wait for user confirmation">
    The skill waits for the user to say "done" or similar.
  </Step>
  <Step title="Verify">
    After confirmation:

    ```bash
    $B cookies
    ```

    Show the user a summary of imported cookies (domain counts).
  </Step>
</Steps>

## Notes

- **First import per browser** may trigger a macOS Keychain dialog — click "Allow" or "Always Allow"
- **Cookie picker is served on the same port** as the browse server (no extra process)
- **Only domain names and cookie counts are shown** in the UI — no cookie values are exposed
- **The browse session persists cookies** between commands, so imported cookies work immediately

## Usage

```bash
# Interactive picker
/setup-browser-cookies

# Direct import (skip UI)
/setup-browser-cookies github.com

# Specify browser
/setup-browser-cookies github.com --browser chrome

# Multiple domains
/setup-browser-cookies staging.myapp.com
/setup-browser-cookies github.com
```
